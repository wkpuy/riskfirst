2.3 Earnings Window — ความเสี่ยงพิเศษ

งบการเงินรายไตรมาสคือ "ความเสี่ยงที่ไม่อาจคาดเดาได้" — ราคาอาจ Gap ขึ้นหรือลง 10–30% ในคืนเดียว ทำให้ Stop Loss ไม่ทำงาน (เกิด Slippage รุนแรง)

🚫 0–5 วันก่อนงบ — ห้ามเข้า

ความเสี่ยงสูงสุด — อย่าเข้าซื้อ position ใหม่ ถ้ามี position อยู่แล้วพิจารณา reduce size หรือตั้ง Stop กว้างขึ้น (3.0×ATR แทน 2.5×ATR)

⚠️ 6–14 วันก่อนงบ — ระวัง

IV (Implied Volatility) เริ่มสูงขึ้น ถ้าเข้าได้ให้ใช้ Stop กว้างกว่าปกติ หรือลด position size ลงครึ่งหนึ่ง

🎯 3–45 วันหลังงบ — Post-Earnings Setup

หนึ่งใน High-Probability Setup ที่ Minervini แนะนำ — ถ้าหุ้น SEPA ผ่านครบหลังงบออก แสดงว่า "งบดีและตลาดชอบ" Base ที่เกิดหลังงบมักแข็งแกร่งมาก

ระบบประมาณวันงบ = วันงบล่าสุด + 90 วัน (ไม่ใช่วันจริง อาจคลาดเคลื่อน ±2–3 สัปดาห์)

3. Risk Management

"ผู้ชนะไม่ใช่คนที่ถูกมากที่สุด แต่คือคนที่ผิดได้โดยไม่เจ็บปวดมาก" — Ed Seykota

3.1 ATR Stop Loss

ATR (Average True Range) วัด "ความผันผวนปกติ" ของหุ้นแต่ละตัว ทำให้ Stop Loss เหมาะกับหุ้นนั้นๆ ไม่ใช่ใช้ % เดียวกันทุกตัว

// True Range ของแต่ละวัน TR = max( High − Low, // Range ภายในวัน |High − PrevClose|, // Gap ขึ้น |Low − PrevClose| // Gap ลง ) // ATR-14 ด้วย Wilder's Smoothing (ไม่ใช่ Simple Average) ATR[0] = Average(TR, 14) // seed ด้วย SMA 14 bar แรก ATR[i] = (ATR[i-1] × 13 + TR[i]) / 14 // Stop Loss stop = entry − (ATR × 2.5) stop = max(stop, entry × 0.01) // floor: ต่ำสุด 1% ของราคา

ทำไม Wilder's Smoothing ไม่ใช่ Simple Average? Wilder's EMA ให้น้ำหนักกับ ATR วันก่อนมากกว่า (×13/14) ทำให้ผลเรียบกว่า ตอบสนองต่อ spike ช้ากว่า และตรงกับ ATR ที่นักเทรดสถาบันและ Minervini ใช้จริง Simple Average จะ "กระโดด" มากเกินไปหลัง Volatility spike

3.2 1% Risk Rule — Position Sizing

กฎเหล็กที่ว่า "ในแต่ละ trade จงเสี่ยงไม่เกิน 1% ของพอร์ต" — ถ้าทุน $1,000 และ Stop ถูก Trigger จะขาดทุนไม่เกิน $10 แต่ละ trade

riskPerTrade = accountSize × (riskPct / 100) // เช่น $1,000 × 1% = $10 riskPerShare = entryPrice − stopPrice // ระยะห่างระหว่าง Entry กับ Stop shares = riskPerTrade / riskPerShare // ปัดลง (ถ้าไม่ใช้ Fractional) positionValue = shares × entryPrice positionPct = (positionValue / accountSize) × 100
💡 ทำไม 1% ไม่ใช่ 5% หรือ 10%?

สมมุติ win rate 50% (ถูก 1 ผิด 1) และ R/R = 2:1 ถ้าเสี่ยง 1% ต่อ trade: ขาดทุน 10 trade ติดกัน (โอกาส 0.1%) ก็เสียแค่ 10% ของพอร์ต แต่ถ้าเสี่ยง 10% ต่อ trade: ขาดทุน 3 trade ติดกัน (โอกาส 12.5%) ก็เสียไป 27% แล้ว — 1% คือ "ประกันความอยู่รอด" ในระยะยาว

⚠️ Alert: ถ้า position value เกิน 20% ของพอร์ต ระบบจะเตือน — หมายความว่า stop ใกล้มากเกินไป ควรขยาย stop หรือลด risk % ลง

3.3 R-Multiple — วัด Risk/Reward

R-Multiple คือหน่วยมาตรฐานที่ Van Tharp นิยมใช้ในการวัดผลการเทรด แทนที่จะพูดว่า "กำไร $500" ให้พูดว่า "กำไร 2R" — เพราะมันบอกว่าได้กำไรเท่ากับ 2 เท่าของ Risk ที่เสี่ยงไป

1R = riskPerTrade // ความเสี่ยงต่อ trade = 1 หน่วย R realizedR = (sellPrice − buyPrice) / (buyPrice − stopPrice) // ตัวอย่าง: Buy $100, Stop $90, Sell $120 riskPerShare = $100 − $90 = $10 (= 1R) gainPerShare = $120 − $100 = $20 realizedR = $20 / $10 = 2.0R ✅ // Target selection: 2R target = entry + (risk × 2) 3R target = entry + (risk × 3)
📐 ทำไมต้องเลือก Target ก่อนเข้า?

การรู้ R/R ก่อนเข้า trade ช่วยกรอง trade ที่ไม่ดีออกไป ถ้า 52w High อยู่ต่ำกว่า 2R target แสดงว่า Upside จำกัดมาก — อาจไม่คุ้มที่จะเสี่ยง Minervini แนะนำ min R/R = 3:1 (เสี่ยง 1 ได้ 3)

3.4 Portfolio Heat

Portfolio Heat คือ "ความเสี่ยงรวม" ของทุก position ที่เปิดอยู่ในขณะนี้ เปรียบเหมือนอุณหภูมิของพอร์ต — ยิ่งสูงยิ่งอันตราย

heatPerPosition = shares × (entryPrice − stopPrice) // max loss ถ้า stop hit totalHeat = Σ heatPerPosition portfolioHeat% = (totalHeat / accountSize) × 100 // เช่น มี 3 position เสี่ยงตัวละ 1% totalHeat% = 1 + 1 + 1 = 3% ✅ ปลอดภัย

⚠️ Max Portfolio Heat แนะนำ: 6–8% ถ้าตลาดพลิกและ Stop ทุก position ถูก Trigger พร้อมกัน จะขาดทุนไม่เกิน 6–8% ในวันเดียว — ยังอยู่รอดได้และฟื้นกลับมาได้

4. Trade Journal

Journal ไม่ใช่แค่บันทึก — มันคือ "data" ที่บอกว่าระบบของคุณทำงานหรือเปล่า

4.1 Performance Metrics

Win Rate

Dashboard
winRate = wins / totalTrades × 100

Win rate ที่ดีคือเท่าไหร่? ไม่ใช่ 70–80% เสมอไป — ถ้า R/R = 3:1 แค่ win rate 30% ก็ยังมี Edge ดู Profit Factor ร่วมด้วยเสมอ

Profit Factor

สำคัญที่สุด
profitFactor = grossProfit / grossLoss > 1.0 = มี Edge (ทำกำไรในระยะยาว) > 1.5 = ดี > 2.0 = ดีมาก

Profit Factor รวม Win Rate และ R/R เข้าไว้ด้วยกัน เป็นตัวเลขเดียวที่บอกว่าระบบคุณ "มี Edge หรือเปล่า" ถ้าต่ำกว่า 1.0 = ระบบขาดทุนในระยะยาว ไม่ว่าจะ win rate สูงแค่ไหน

Avg Realized R/R

Dashboard
avgRR = Σ realizedR / closedTrades realizedR[i] = (sellPrice − buyPrice) / (buyPrice − stopPrice)

ค่าเฉลี่ย R-Multiple ที่ realize จริงในทุก trade ถ้า Avg R/R < 1.0 แสดงว่าคุณกำลัง "ปล่อยให้ผู้แพ้วิ่งและตัดผู้ชนะเร็วเกินไป" ซึ่งเป็นข้อผิดพลาดทางจิตวิทยาที่พบบ่อยมากที่สุด

Expectancy (ค่าคาดหวัง)

Van Tharp
expectancy = (winRate × avgWin) − (lossRate × avgLoss) // ตัวอย่าง: Win 50%, Avg Win $200, Avg Loss $100 expectancy = (0.50 × $200) − (0.50 × $100) = $50 ต่อ trade ✅

Expectancy บอกว่า "โดยเฉลี่ยแล้วจะได้หรือเสียเท่าไหร่ต่อ trade" ถ้าเป็นบวก = ระบบมี positive expected value = มี Edge ทางสถิติ

Benjamin Graham อุปมาว่าตลาดหุ้นเหมือน "Mr. Market" ที่เสนอราคาซื้อขายทุกวัน บางวันเขา Optimistic เสนอราคาสูงมาก บางวัน Pessimistic เสนอราคาต่ำมาก — หน้าที่ของคุณคือรอให้เขาเสนอราคาที่ต่ำกว่ามูลค่าที่แท้จริงมากพอ แล้วค่อยซื้อ ไม่ใช่ตาม mood ของเขา

5.1 VI Quality Score (0–10 คะแนน)

ระบบแยก Growth Stock (EPS โต > 25% หรือ Revenue โต > 20%) ออกจาก Value/Defensive Stock แล้วใช้เกณฑ์คนละชุด

10 เกณฑ์ VI Quality Score
1
ROE ≥ 15% (Return on Equity)

ผลตอบแทนต่อทุนผู้ถือหุ้น — Buffett ให้ความสำคัญมาก บริษัทที่ ROE สูงสม่ำเสมอ = มี Competitive Moat แท้จริง

ROE = Net Income / Shareholders' Equity × 100%
2
ROE ≥ 10% (ขั้นต่ำ)

เกณฑ์รองเพื่อให้คะแนนบางส่วนแม้ไม่ถึง 15% — บริษัทที่ ROE ต่ำกว่า 10% ถือว่าการบริหารทุนไม่มีประสิทธิภาพ

3
Revenue Growth > 0%

ยอดขายต้องโต — ธุรกิจที่หดตัวด้านรายได้มักสะท้อนปัญหาพื้นฐาน แม้กำไรอาจยังดูดีจากการลดต้นทุน

4
EPS Growth > 0%

กำไรต่อหุ้นต้องโต — ถ้า Revenue โตแต่ EPS ลดแสดงว่า Margin กำลังถูกกดดัน หรือบริษัทออกหุ้นใหม่ Dilute ผู้ถือหุ้น

5
P/E < 30 (Value) · PEG < 1.5 (Growth)

Value: ไม่จ่ายแพงเกินไปเทียบกับกำไร · Growth: Peter Lynch แนะนำ PEG < 1.5 เพราะ P/E สูงของ Growth Stock ต้องดูเทียบกับ Growth Rate

PEG = P/E Ratio / EPS Growth% // < 1.5 = fair for growth
6
P/E > 0 (มีกำไร)

บริษัทต้องมีกำไรจริง — หุ้นที่ P/E ติดลบหรือไม่มีกำไรเป็นการเก็งกำไร ไม่ใช่การลงทุนเชิงคุณค่า

7
P/B < 5 (Value) · ROE > P/B (Growth)

Value: ราคาไม่ควรแพงกว่ามูลค่าทางบัญชีมากเกินไป · Growth: ถ้า ROE > P/B แปลว่าบริษัทสร้างผลตอบแทนได้มากกว่าที่ตลาดจ่าย = ยังคุ้ม

8
Beta (ความผันผวน)

Value: Beta < 1.2 = ผันผวนน้อยกว่าตลาด เหมาะกับการถือยาว · Growth: Beta อาจสูงได้ ระบบไม่หักคะแนนเพราะเป็นเรื่องปกติของ Tech

9
ROA > 5% (Return on Assets)

ใช้สินทรัพย์สร้างกำไรได้ดีแค่ไหน บริษัท Asset-heavy (โรงงาน โรงแรม) อาจมี ROA ต่ำตามธรรมชาติ ต้องดู Industry เปรียบเทียบด้วย

ROA = Net Income / Total Assets × 100%
10
ตำแหน่งใน 52w Range

Value: ราคาอยู่ใน 40% ล่างของ 52w Range (ใกล้ Low = โอกาส MOS สูง) · Growth: อยู่ใน 40% บน (ใกล้ High = Momentum ยังดี)

posInRange = (price − low52) / (high52 − low52)

5.2 Fair Value — มูลค่าที่แท้จริง คืออะไร?

Fair Value (หรือ Intrinsic Value) คือ มูลค่าที่แท้จริงของบริษัท ที่ประเมินจากการวิเคราะห์พื้นฐาน ไม่ใช่ราคาตลาดที่ซื้อขายกันอยู่ ราคาตลาดจะวนเวียนอยู่รอบ Fair Value ขึ้นบ้างลงบ้างตาม sentiment — หน้าที่ของ VI คือซื้อเมื่อราคาตลาด ต่ำกว่า Fair Value มากพอ

💡 แอปไม่ได้คำนวณ Fair Value ให้ — เพราะอะไร?

Fair Value ขึ้นอยู่กับ assumptions ส่วนตัว เช่น growth rate ที่คาดหวัง, WACC ที่ใช้, จำนวนปีที่จะถือ — สองคนวิเคราะห์บริษัทเดียวกันอาจได้ Fair Value ต่างกันได้ 30–50% แอปให้คุณใส่ตัวเลขที่คุณ เชื่อ แล้วช่วยคำนวณว่าควรซื้อเท่าไหร่เมื่อเทียบกับราคาตลาด

วิธีประเมิน Fair Value ที่นิยมใช้

1️⃣
Analyst Consensus Target Price (ง่ายสุด)

ดูจาก Bloomberg, Reuters, Seeking Alpha, หรือ brokerage research — เป็นค่าเฉลี่ยการประเมินของนักวิเคราะห์หลายคน เหมาะสำหรับผู้เริ่มต้น

แหล่ง: Yahoo Finance (Analysts' Price Target), TradingView
2️⃣
Forward P/E Method (เร็ว ใช้บ่อย)

ใช้ P/E ที่ "ยุติธรรม" ของ sector คูณกับ EPS ที่คาดว่าจะได้ปีหน้า (Forward EPS)

Fair Value = Fair P/E × Forward EPS // ตัวอย่าง: บริษัท Consumer อิ่มตัว fair P/E ≈ 15–20x // EPS next year = $5 → Fair Value = 18 × $5 = $90
3️⃣
Graham Number (Benjamin Graham)

สูตรง่ายของ Graham ที่ใช้ EPS และ Book Value — เหมาะกับ Value Stock ไม่เหมาะกับ Growth

Graham Number = √(22.5 × EPS × Book Value per Share) // ตัวอย่าง: EPS = $4, Book Value = $25 // Graham Number = √(22.5 × 4 × 25) = √2,250 ≈ $47.4
4️⃣
DCF — Discounted Cash Flow (แม่นสุด ยากสุด)

ประมาณ Free Cash Flow ทุกปีในอนาคต แล้ว discount กลับมาเป็นมูลค่าปัจจุบัน ต้องการ assumptions มากที่สุด

Fair Value = Σ FCF_t / (1+r)^t + Terminal Value / (1+r)^N ÷ shares outstanding // r = Discount Rate (WACC ~8-10%), N = ปีที่ประมาณ (5-10 ปี) // Terminal Growth ≈ 2-3% (GDP long-term)

⚠️ DCF sensitive มาก — growth rate เปลี่ยน 2% → Fair Value เปลี่ยนได้ 30–50% เสมอ run 3 scenario: Bear / Base / Bull

✅ แนวทางปฏิบัติ: เลือกวิธีให้เหมาะกับหุ้น
Growth Stockใช้ Forward P/E หรือ PEG-implied value — Graham Number ไม่เหมาะเพราะ Book Value ต่ำ
Value/Dividendใช้ Graham Number + DCF ร่วมกัน + เทียบ Analyst Target
ไม่แน่ใจใช้ Analyst Target เป็น starting point แล้วปรับตาม VI Score ที่แอปคำนวณ

5.3 Margin of Safety — ส่วนเผื่อความปลอดภัย

Benjamin Graham เสนอแนวคิดนี้ใน The Intelligent Investor (1949) — "ซื้อเฉพาะเมื่อราคาต่ำกว่ามูลค่าอย่างมีนัยสำคัญ" MOS คือ "ส่วนลด" จาก Fair Value ที่คุณประเมิน

MOS% = (FairValue − CurrentPrice) / FairValue × 100 // ตัวอย่าง: Fair Value $100, ราคาปัจจุบัน $70 MOS = (100 − 70) / 100 × 100 = 30% ✅ ถึงเกณฑ์ซื้อ
🟢
MOS > 30% — BUY LARGE (100% ของ Max Position)
ราคาถูกมาก มีส่วนเผื่อความปลอดภัยสูง Graham แนะนำ 30–50%
🔵
MOS 20–30% — BUY MEDIUM+ (70%)
ราคาดี มี MOS พอสมควร เข้าได้แต่ไม่เต็มมือ
🔵
MOS 10–20% — BUY MEDIUM (50%)
ราคายังพอรับได้ ลงทุนครึ่งหนึ่งของแผน
🟡
MOS 5–10% — BUY SMALL (30%)
เริ่มต้นน้อยๆ เพื่อ tracking รอราคาดีขึ้น
🔴
MOS < 0% — TOO EXPENSIVE (0%)
ราคาแพงกว่า Fair Value แล้ว รอหรือข้าม

5.4 VI Position Sizing

VI ใช้ Fixed Allocation ไม่ใช่ 1% Risk Rule เพราะไม่มี Stop Loss แบบ Trader — แทนที่จะถือ Stop เป็นตัวกำหนด ใช้ "% สูงสุดต่อหุ้น 1 ตัว" แทน

// Rule 1: Fixed Allocation (Hard Ceiling) maxPosition = portfolio × (allocPct / 100) // เช่น $10,000 × 10% = $1,000 max per stock // Rule 2: MOS Weighting (ลด/เพิ่มตาม MOS) buyAmount = maxPosition × mosWeight // mosWeight: 0.20 → 1.00 ตาม MOS tier shares = buyAmount / currentPrice

ทำไม 2 Rules ไม่ใช่ 1? Rule 1 ป้องกัน Concentration Risk (ไม่ถือหุ้นตัวเดียวเกิน 10%) · Rule 2 ทำให้ซื้อมากเมื่อถูก ซื้อน้อยเมื่อแพง — ผลลัพธ์คือ Dollar-Cost Averaging ที่ "똑똑" กว่า เพราะน้ำหนักตาม Valuation

📊 Portfolio Reallocation Logic

ระบบเปรียบเทียบราคาปัจจุบัน (จาก Sync Prices) กับ Fair Value ที่คุณตั้งไว้ แล้วแนะนำว่าควรทำอะไร

TRIM/SELLราคา > Fair Value × 1.05 (แพงเกินไป 5%)
ADD MOREMOS ยังเหลือ ≥ 20% (ยังถูกอยู่)
HOLDMOS 5–20% (ใกล้ Fair Value รอดูทิศทาง)
REVIEWMOS < 5% (จ่อ Fair Value แล้ว ทบทวน thesis)
